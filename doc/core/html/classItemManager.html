Detailed Description

High level interface providing CRUD operations on concrete Items.

This class is used to store and navigate through an Item hierarchy (see Item and derivated classes for further informations).

A part of the interface provides factory methods to create the different kinds of Items, the second one provides high-level methods on Item tree management.

Constructor & Destructor Documentation

ItemManager::ItemManager ( )

Constructs a new ItemManager.

The root representing the managed tree is created with the generic name _root_.

Note:
The root Item is the only one in the hierarchy to have a parent setted to nullptr.
ItemManager::ItemManager ( const ItemManager &  itemManager)

Constructs an ItemManager from the given ItemManager.

The Item's tree root pointer is setted as the root of the constructed ItemManager.

Warning:
The root of the constructed ItemManager is not a copy of the given one. The two ItemManager share and manage the same hierarchy.
ItemManager::~ItemManager ( )

Deletes the ItemManager and the Item representing the herarchy tree.

All the Item contained in the tree are recursively deleted (see Folder::~Folder for further informations).

Warning:
In case of root sharing between ItemManagers, this deletion invalidates the other ItemManagers referencing the root.

Generic find Item method.

Searches the Item with the given name and type in all or part of the Item tree.

Parameters:
itemNamethe name of the wanted Item.
itemTypethe type of the wanted Item.
parentFolderthe Item to start the research from. It allow research in subtrees, wich is faster than a complete research.
Returns:
the Item pointer if the wanted Item is in the Item tree, nullptr otherwhise.
Note:
The Item is returned through the global interface Item, to get concrete Item types object use the appropriated find method. You can also use this method with the appropriated cast method.
If ANY_TYPE is given as itemType all the items corresponding to the other parameters are returned.
